Frequently Asked Questions

Where’s the colorized notation?

▼

After you click “Render,” the color notation appears below on the webpage. A button labeled “Download All as ZIP” also appears for you to download the rendered notation to your local device. The ZIP folder contains both black-and-white and colorized versions of each song page.

Why is the score title not showing?

▼

If Song & Artist is not provided by the user then best attempt to extract these details from the uploaded file is performed. In some cases these details were not provided in the original file. Fill in the optional Song & Artist textbox if this occurs (or if the details are lacking/incorrect).

What song formats can I use?

▼

Currently the tool includes a drop-down that lists three (3) acceptable file types:

  • MusicXML, Compressed (.mxl) — This is the standard file type for songs downloaded from sites like MuseScore. These contain the XML file describing the musical score and any related assets such as images, fonts, etc.

  • MusicXML, Plain (.musicxml, .xml) — Just the XML file describing the musical score, no related assets included.

  • MEI (.mei) — This text-based file format is generated by tools like MuseScore, Verovio, and mei-friend.

Where can I find songs in MusicXML format?

▼

A growing number of sites make downloadable sheet music in MusicXML format, or in formats that can be converted to MusicXML.


This site lists where you can download sheet music in MusicXML or compatible formats.

Can I colorize my own original songs?

▼

Yes. This rendering tool allows you to colorize any notation created using the supported formats. So any song you compose with a tool like MuseScore, for example, can be converted to ColorMusic®.

Are the colors used random or just to look pretty?

▼

No. The colors highlight the music theory used to compose each song, based on the ColorMusic® system.

Who created this tool?

▼

As a collaborative project with Mike George, this ColorMusic Notation Tool was created by Jake Kirsch using Python (FastAPI) and JavaScript for development, Render for web hosting, and Google Cloud Storage for file storage. He is a data engineer based in Atlanta.

Do I need to interpret for key signatures and accidentals in the composition after colorizing?

▼

No, and that's the beauty of the ColorMusic® system. The color/shape you see is the note you play, no additional interpretation required. The symbols for key signature and accidentals have been preserved in the rendering for reference but not required for playing.

How do I print the sheet music?

▼

Click the “Download All as ZIP” button, then print each colorized page using your printer dialog box. You may need to adjust the print size settings to fit the sheet music within the dimensions of your pages.

▼

All files are temporarily stored in a private bucket in Google Cloud Storage with max lifetime of approximately 1-day.
